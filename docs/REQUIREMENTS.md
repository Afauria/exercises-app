# 题库练习 Web 应用 — 需求说明

> 本文档描述当前仓库实现范围内的**产品目标、功能边界、数据与部署约定**，供复用模板、二次开发或交接时使用。实现细节以源码为准。

---

## 1. 产品定位

| 项 | 说明 |
|----|------|
| 名称 | 题库练习（静态 SPA） |
| 形态 | 纯前端，无自建后端；构建产物可部署到任意静态托管（含 GitHub Pages） |
| 终端 | 现代浏览器（含手机浏览器）；无需 App 商店、Xcode 或模拟器 |
| 题库 | 内置：构建期由 TXT 生成 `bank.json`；可选：用户导入 TXT 覆盖（存 `localStorage`） |

---

## 2. 用户与场景

- **学习者**：按节练习、模拟考试、查看错题与收藏、看简单统计。
- **运维/发布者**：`npm run build` 产出 `dist/`，上传或 CI 推送到 Pages。

---

## 3. 功能需求

### 3.1 全局布局

- 顶栏：当前模块标题 + **显示/隐藏全部答案**（眼睛）+ **数据与题库**（菜单入口）。
- 底栏四个主导航（Tab）：**练习 / 考试 / 错题 / 统计**；无独立「设置」Tab。
- 默认进入 **练习**（`/` 重定向到练习列表）。

### 3.2 题库加载

- 启动时请求 **`import.meta.env.BASE_URL` + `bank.json`**（与 Vite `base` 一致）。
- 若存在 **`localStorage` 自定义题库 JSON**，优先使用，不再请求内置 `bank.json`。
- 加载失败时提示错误，并引导通过菜单导入 TXT。
- 练习首页展示 **总题量**（「共 N 题」）、**分节入口**（按题号序 `ordinal` 每节 50 题，见 §5）、**随机一节**、**搜索**（题号或题干关键词，结果可进入单题或小列表练习）。

### 3.3 练习模式

- 从分节或搜索进入 **做题页**，题目列表由路由 `state` 传入（刷新会丢会话，符合当前实现）。
- **上一题 / 下一题 / 完成**：允许**不选答案**即下一题（跳过）。
- **离开当前题**（下一题、完成、返回列表）时写入统计：
  - 已选答案：记一条练习记录（对/错）；错则 **错题 ID 列表** 增加该题 `id`。
  - 跳过：记 **错误**、`userChoice` 为 `(跳过)`，并加入错题本。
- **上一题**：若回到**尚未选题**的题目，清除该题在本轮中的「已落库」标记，允许返回后补答（可能产生多条练习记录）。
- 单题可 **收藏 / 取消收藏**（与错题分区展示）。
- 题目展示支持单选（A–D）与判断（正确/错误）；可与全局「显示答案」叠加。

### 3.4 考试模式

- **选节**（与练习相同的节划分）、**时长（分钟）**、**题量**（不超过本节可用题数）；从本节随机抽题。
- 倒计时；**下一题 / 完成**；允许不答即下一题；未答题计为不得分。
- **交卷**提前结束并计分。
- 结果页展示得分；考试记录写入本地。

### 3.5 错题与收藏

- **需要巩固的题目**：来自错题 ID 列表，可跳转练习、可移除。
- **收藏**：列表展示，可取消收藏、可跳转练习。

### 3.6 统计

- **练习正确率**：基于本地练习记录聚合。
- **题库总量**：当前内存题库题目数。
- **近 30 日节奏**：按日汇总练习记录条数。
- **考试记录**：最近若干条会话摘要。
- 写入 `localStorage` 后通过自定义事件 **`app-data-updated`** 刷新本标签页展示（`storage` 事件仅跨标签页）。

### 3.7 数据与题库菜单

- **清空全部数据**：清除错题、收藏、练习记录、考试记录、全局显隐答案；**不删除**自定义题库 JSON（若曾导入）。
- **导入内置题库**：清除自定义题库缓存并整页刷新，恢复内置 `bank.json`。
- **导入本地 TXT**：解析成功后写入自定义题库并持久化；体积受浏览器 `localStorage` 限制。

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 离线/弱网 | 部署后主要资源本地缓存；首访需拉取 `bank.json` 与静态资源 |
| 性能 | 大题库以 JSON 形式加载；构建期生成 JSON，避免运行时解析整份 TXT |
| 可部署性 | 纯静态文件；GitHub Pages 项目站路径为 `/<仓库名>/` |
| 路由 | **Hash 路由**（`#/practice` 等），避免刷新子路径 404 |
| 可测试性 | `npm run verify`：`tsc` + 生产构建 + Playwright（题库加载、答题、清空数据等） |

## 5. 数据与持久化

### 5.1 存储介质

- 仅 **`localStorage`**（前缀 `exam_web:`），无数据库。

### 5.2 键与含义（逻辑）

| 逻辑键 | 用途 |
|--------|------|
| wrong_ids | 错题题目 `id` 列表 |
| favorite_ids | 收藏题目 `id` 列表 |
| practice_records | 练习作答记录（含 `questionId`、`correct`、`userChoice`、`at`） |
| exam_sessions | 考试场次（总分、正确数、用时、结束时间等） |
| reveal_all | 全局是否显示答案 |
| custom_bank_json | 用户导入的整包题库 JSON 字符串（可选） |

### 5.3 题目与题库结构（JSON）

- `Question`：`id`（构建内顺序）、`ordinal`（源文件题号）、`stem`、`qtype`（`choice` \| `boolean`）、`answer`、`explanation`、`options`。
- `BankDocument`：`bankName`、`questions[]`、可选 `parseErrors`。

### 5.4 分节规则

- 常量 **`PAGE_SIZE = 50`**。
- 第 `k` 节（从 0 起）：`ordinal ∈ [k*50+1, (k+1)*50]`。
- 节数量由 **`ceil(maxOrdinal / 50)`** 决定。

## 6. 题库来源与构建流水线

1. **源文件**：`assets/bundled-question-bank.txt`（约定格式由 `questionBankParser` 解析）。
2. **脚本**：`scripts/build-bank-json.ts`（`prebuild` / `dev` 前执行）→ 输出 **`public/bank.json`**。
3. **Vite 构建**：将 `public/` 内资源拷入 `dist/`（含 `bank.json`）。
4. **GitHub Pages 辅助**：构建后复制 **`dist/index.html` → `dist/404.html`**，减轻误访问非 hash 路径时的裸 404（主要依赖仍建议 Hash 路由）。

## 7. 路由一览（Hash 内路径）

| 路径 | 说明 |
|------|------|
| `/` → `/practice` | 重定向 |
| `/practice` | 练习首页 |
| `/practice/quiz` | 做题（依赖 `location.state.questions`） |
| `/exam` | 考试设置 |
| `/exam/quiz` | 考试中（依赖 `state`） |
| `/exam/result` | 考试结果（依赖 `state`） |
| `/wrong` | 错题与收藏 |
| `/stats` | 统计 |

## 8. 部署与 `base` 路径

- **Vite `base`** 必须与线上 URL 前缀一致：`https://<user>.github.io/<仓库名>/` → `base = '/<仓库名>/'`。
- **解析顺序**：环境变量 **`VITE_BASE`** → **`GITHUB_REPOSITORY`**（CI 自动注入，取 `repo` 段）→ 本地回退（当前默认为 `/ai_test/`，可按仓库改名）。
- **Playwright**：`E2E_BASE_PATH` 与上述路径对齐；可用 **`E2E_BASE`** 覆盖。

## 10. 已知限制与迭代建议

- **做题页 `state`**：直接刷新可能丢失题目列表，需从练习首页重新进入（若需可改为 URL 参数或 sessionStorage）。
- **错题本**：答对后不会自动从错题移除（可迭代「答对移出」策略）。
- **大题库**：单文件 `bank.json` 过大时可拆分为按节懒加载。
- **多题库**：当前仅「内置 + 可选一份自定义覆盖」；扩展需数据模型与 UI。
- **国际化**：界面为中文；复用到其它语言需抽文案。
